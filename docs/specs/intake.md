## §1. Brief 핵심 사실 (확정)

- **앱 컨셉**: 일반 뉴스앱으로 위장된 1:1 비밀 채팅. "비밀요원 놀이" 용 SaaS
- **사용자 유형**: 단일 — 에이전트 (이메일+비밀번호 가입자). admin/타 권한 없음
- **페어링 모델**: ID 검색 → 요청 → 수락/거부 → PAIRED. 1 사용자 = 1 active pair. 변경 시 기존 페어 DISCONNECTED
- **위장 진입**: 메인 뉴스 피드의 특정 기사들을 특정 순서로 탭 → 채팅 화면 전환
- **위장 알림**: 메시지 도착 시 "📰 새 뉴스 — ..." 형식 푸시. 탭하면 뉴스 앱 열림. 시퀀스 입력 후만 메시지 확인
- **첨부**: 사진 / 파일 전송 가능 (OS sharesheet / 파일 picker)
- **비-목표 (v1)**: 다국어, 웹/네이티브 분기, 그룹 채팅, E2E 암호화
- **성공 지표 (6개월)**: 페어 3쌍 (친구 6명), 일일 ≥10,000회 비밀 채팅, 가족 위장 테스트 통과
- **보안**: bcrypt 비번 해싱 + 채팅 메시지 서버측 암호화 (E2E v1 밖)
- **인프라**: Docker self-host, 무료/저비용

## §3. 사용 시나리오 (확정)

1. **위장 진입 (Happy)**: 뉴스 피드에서 5 → 3 → 1 → 7 순으로 기사 탭 → 채팅 전환
2. **위장 알림**: 메시지 도착 → "📰 새 뉴스" 푸시 → 탭 → 뉴스 앱 → 시퀀스 입력 → 메시지 확인
3. **첨부 전송**: 채팅에서 사진/PDF 첨부

## §가정 (intake 미답변 갭 — 가정값 채택)

다음 3개 갭은 brief 에 명시 없음. SV 가 brief context + 메모리 + 일반 보안/위장 원칙 기반으로 가정 채택. **PM/Arch/UX 는 다음 가정을 그대로 따르되, 채택 시 산출물 끝 "전제" 섹션에 가정 출처 명시.**

### G-1. 시퀀스 메커니즘

- **시퀀스 등록 주체**: **사용자 본인** 이 앱 가입 후 최초 1회 등록. 변경은 설정 화면에서 가능 (단, 현재 시퀀스 재입력 인증 필요)
- **"기사 N번" 정의**: **뉴스 피드 현재 노출 순서 (top N)** 기준. 즉 매번 뉴스 API 가 다른 기사를 내려주더라도 "위에서 N번째 카드" 가 N
- **시퀀스 길이**: 4~6개 (보안과 사용성 절충)
- **시퀀스 분실 복구**: 이메일 인증 기반 reset (현재 시퀀스 무력화 후 재등록). reset 후 안전을 위해 페어는 유지하되 채팅 히스토리는 보존
- **잘못된 시퀀스 입력 시**: 뉴스 앱이 정상 동작하는 것처럼 보이게 (위장 유지). 시퀀스 실패 카운터 무한 — rate limit 만 적용 (1초 내 같은 기사 재탭은 무시)

### G-2. 위장 유지 깊이

- **백그라운드 진입 시**: 채팅 화면이라도 **항상 뉴스 피드 화면으로 강제 복귀**. 앱 재진입 시 위장 깨짐 0%
- **앱 종료 후 재시작**: 자동 로그아웃 X, 단 채팅 화면은 보이지 X. 항상 뉴스 피드부터
- **채팅 히스토리 보존**: **90일** (이후 자동 삭제). 90일 이상 안 쓰는 메시지는 위험 자산
- **페어 해제 (DISCONNECTED) 시**: 양측 메시지 **즉시 삭제** (위장 강화). 첨부 파일도 즉시 S3/스토리지에서 삭제
- **앱 백그라운드 ↔ foreground 빠른 전환 시**: 위장 화면 (뉴스 피드) 가 보장. 5초 이상 백그라운드면 자동 채팅 화면 종료

### G-3. 첨부파일 & 위장 알림

- **저장**: **MinIO** (S3 호환, Docker self-host 가능. NestJS aws-sdk 로 동일 API)
- **최대 크기**: 이미지 ≤ 10MB, PDF/일반파일 ≤ 25MB
- **허용 mime type**: `image/jpeg`, `image/png`, `image/webp`, `application/pdf`. 그 외는 거부
- **보존 기간**: 메시지 보존 정책과 동일 — 90일 (메시지 삭제 시 첨부도 cascade 삭제)
- **위장 알림 카피 (첨부 종류별)**:
  - 텍스트: `📰 새 뉴스 — {랜덤 가짜 헤드라인}`
  - 사진: `📰 새 뉴스 사진 — {랜덤 가짜 헤드라인}`
  - PDF/파일: `📰 새 뉴스 자료 — {랜덤 가짜 헤드라인}`
- **랜덤 가짜 헤드라인 풀**: 서버에 미리 50개 한국어 뉴스 카피 풀 보관. 발송 시 임의 선택
