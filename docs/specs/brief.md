# Brief — agentNews (비밀요원 채팅 뉴스앱)

## 1. 한 줄 정의

겉으로는 일반 뉴스 앱처럼 보이지만, 특정 순서로 기사를 탭하면 비밀 1:1 채팅 모드로 진입하는 친구들끼리의 "비밀요원 놀이" 용 위장 SaaS.

## 2. 풀려는 문제

- 친구들과 "비밀요원 / 첩보" 컨셉의 놀이를 하려면, 일상적으로 쓸 수 있는 위장 앱이 있어야 몰입이 된다
- 일반 채팅 앱 (카카오톡, 디스코드 등) 은 "비밀" 느낌이 안 남
- 폰을 다른사람이 훔쳐가서 봐도 "그냥 뉴스 애플리케이션" 으로 보여야 함
- 알림조차 "새 메시지 1건" 이 아니라 "새 뉴스 기사" 로 위장돼야 함
- 사진/파일 전송도 가능해야 작전 자료 (가짜 첩보 사진, 미션 파일) 를 주고받을 수 있음

## 3. 핵심 사용자 유형

- **에이전트 (회원)** — 이메일+비밀번호로 가입한 사용자. 뉴스 열람 + 비밀 시퀀스 입력 시 채팅 가능

**페어링 모델**: 에이전트끼리 **사용자 ID 검색 → 페어링 요청 발송 → 상대방 수락/거부 → PAIRED**. admin 사전 설정 X. 한 사용자 = 1 active pair (변경 시 기존 페어 DISCONNECTED 후 새 요청).

## 4. 핵심 사용 시나리오

1. **위장 진입 (Happy)**: 에이전트 A 가 메인 뉴스 피드에서 1, 3, 5, 7번 기사가 보이는 상태에서 **5 → 3 → 1 → 7** 순으로 탭. 화면이 비밀 채팅창으로 전환. 페어 (에이전트 B) 와 1:1 대화.
2. **위장 알림**: 에이전트 B 가 메시지를 보내면, A 의 폰에 "📰 새 뉴스 — '00 정상회담 합의 도출'" 형식 푸시. 탭하면 뉴스 앱이 열리고 (위장 유지), 시퀀스 입력 후에만 실제 메시지 확인 가능.
3. **첨부 전송**: 채팅 모드에서 사진 (작전 사진) 또는 파일 (미션 PDF) 전송. OS sharesheet / 파일 picker 사용.

**비-목표 (v1 범위 밖)**:
- 다국어 (한국어만)
- 네이티브 앱 외 (RN 단일)
- 그룹 채팅 (1:1 만)
- E2E 암호화

## 5. 출시 후 첫 6개월 성공 지표

- 친구 6명이 페어링 완료 (3쌍)
- 매일 ≥10,000회 비밀 채팅 사용
- 위장 알림 가족 테스트 통과 (가족이 채팅앱인지 못 알아챔)

## 6. 제약 / 비-목표

**제약**:
- 개인적 사용 (친구 6명) — 공식 서비스 아님
- 비밀번호 bcrypt 해싱 + 채팅 메시지 서버측 암호화 (E2E 는 v1 범위 밖)
- 뉴스는 외부 무료 API (public-apis github 리스트 중 선정)
- 무료/저비용 인프라 (Docker self-host 가능)

## 7. 기존 자산 / 참고

**기술 스택**:
- 모바일: **React Native** + TypeScript
- 푸시: FCM
- 백엔드: NestJS
- DB: PostgreSQL 16
- 배포: 백엔드 Docker / docker-compose. 모바일은 **TestFlight (iOS, Apple Dev $99/yr) + Firebase App Distribution (Android)**
- 디자인: Apple Human Interface Guidelines (iOS 우선)
- **위장 강화 이점 (RN 선택 사유)**: URL bar 노출 없음 → 라우트 평문 노출 (`secret-chat-`) 위장 깨짐 자동 해결.

**참고**:
- 디자인 참고: Apple News, Apple Messages
