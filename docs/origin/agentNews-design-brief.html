§1 한 줄 컨셉

겉으로는 일반 뉴스 앱처럼 보이지만, 특정 순서로 기사를 탭하면 비밀 1:1 채팅 모드로 진입하는 친구들끼리의 "비밀요원 놀이" 용 위장 SaaS.

  • 위장 진입: 메인 피드의 1·3·5·7번 기사를 5 → 3 → 1 → 7 순으로 탭 → 채팅 모드 전환
  • 위장 알림: 메시지 도착 시 "📰 새 뉴스 — '00 정상회담 합의 도출'" 형식 푸시
  • 위장 라우트: RN React Navigation Stack — URL bar 없음. 채팅 화면도 ArticleDetailScreen 내부 conditional swap
  • 첨부: 사진/PDF — OS sharesheet 직접 호출

§2 사용자 유형 (1종)

유일 사용자 = agent (회원). 이메일+비밀번호로 가입한 사용자. 페어링은 사용자 ID 검색 → 페어링 요청 → 상대 수락 모델 (admin 사전 설정 아님).

§4 핵심 User Flow (3개)

Flow 1 — 위장 진입 (SCN-002 Happy)

SCR-010 메인 피드 진입. 일반 뉴스 카드 N개 보임 (Apple News 스타일)
사용자가 5번째 기사 탭 → SCR-011 ArticleDetailScreen (일반 모드)
뒤로 → 3번째 → 1번째 → 7번째 기사 차례 탭
4번째 정탑 (7번 기사) 완료 시 unlock_token 발급. 화면이 *같은 ArticleDetailScreen 내부* 에서 채팅 모드로 conditional swap
SCR-020 채팅 — 페어와 1:1 대화 시작 (말풍선 UI, Apple Messages 스타일)

Flow 2 — 위장 알림 tap (SCN-002 Push)

페어가 메시지 보냄 → 백엔드 worker 가 위장 헤드라인 생성
사용자 폰에 푸시: 📰 새 뉴스 — "AI 모델 공개 — 안전성 논쟁 가열" / 연합뉴스 3분 전 형식. data payload 0, notification payload only
사용자가 알림 tap → 앱 열림 (cold start 시 messaging().getInitialNotification())
→ SCR-010 메인 피드 진입 (위장 유지). 자동으로 채팅 화면 이동 X
사용자가 별도로 5-3-1-7 시퀀스 입력해야 실제 메시지 확인

Flow 3 — 첨부 전송 (SCN-006)

SCR-020 채팅 모드. 첨부 아이콘 탭 → OS sharesheet (react-native-image-picker / react-native-document-picker)
사용자가 사진 또는 PDF 선택. 클라이언트가 POST /api/v1/attachments (multipart, status=UPLOADING)
서버: MIME + 매직바이트 3중 검증 → status=AVAILABLE
사용자가 메시지 본문 + 첨부 함께 전송 → 메시지 버블에 inline 미리보기 (이미지 thumbnail / PDF 아이콘)
페어가 받음 → 위장 푸시 → 시퀀스 후 채팅 → 첨부 탭 → OS 기본 뷰어

§6 핵심 컴포넌트

  • NewsCard — 썸네일 (16:9, radius 12) + 제목 + 출처+시간 + 요약 일부. 첫 카드는 hero (큰 이미지). Apple News 풍
  • NewsFeedList — 무한 스크롤. Pull-to-refresh. 카드 사이 12pt
  • ArticleDetailHeader — 큰 이미지 hero + 제목 + 출처 + 발행 시각 + "원문 보기" 버튼. 일반 모드는 여기까지
  • ChatSwap — 같은 ArticleDetailScreen 안에서 unlock_token + PAIRED 시 자동 swap. 위에서부터 채팅 메시지 리스트로
  • MessageBubble — 좌측 peer (회색 #e9e9eb), 우측 me (파랑 #007aff + 흰 글씨). Apple Messages 풍 corner radius 22. 1분 이내 같은 sender 면 grouping
  • AttachmentPreview — 이미지: 200×200 cap, radius 12. PDF: 아이콘 + 파일명 + 크기
  • ChatInputBar — 하단 sticky. 입력 textarea + 첨부 아이콘 + 전송 버튼 (paper plane). 잠금 아이콘 (헤더 우측)
  • PairingSearchInput — 검색창 + 결과 목록 (아바타 + 닉네임 + ID). 페어링 요청 버튼
  • SettingsRow — Apple Settings 스타일. 아이콘 + 라벨 + chevron / toggle / value
  • SequenceTrigger (invisible) — NewsCard 가 받음. tap order 추적은 클라이언트 state. UI 노출 0
  • PushNotificationPreview — mockup 용 위장 알림 (iOS / Android notification banner). title "📰 새 뉴스 — <헤드라인>" / subtitle "<출처> · <시각>". data payload 표기 0

§7 위장 강화 가이드라인 (시각 디자인 단계 enforce)

  • CRITICAL 앱 이름: agentNews 가 아니라 "DailyNews" / "오늘의뉴스" 같은 평범한 뉴스 앱 이름. 앱 아이콘도 신문/N 로고
  • CRITICAL 번들 ID: com.dailynews.app (chat/secret/agent 단어 0)
  • CRITICAL 스크린 이름: ChatScreen 같은 직접 명명 금지. ArticleDetailScreen 내부 swap 패턴
  • HIGH Notification Center preview: 잠금화면에서 헤드라인만 노출. data payload 0
  • HIGH App Switcher preview: 백그라운드 전환 시 채팅 화면 → 일반 뉴스 화면으로 swap (애니메이션 없이)
  • HIGH 스크린샷 차단: Android FLAG_SECURE, iOS 의 경우 OS 차단 불가 → 채팅 모드에서 사용자가 home button 누를 때 화면 blur 처리
  • HIGH iOS Spotlight / Siri Suggest: 메시지 내용 색인 비활성
  • OK 채팅 화면 헤더: 페어 닉네임 대신 "기사 토론 · 3 댓글" 같은 뉴스 어휘 노출 검토 (디자인 옵션)
